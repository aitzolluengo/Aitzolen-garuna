# Captura rápida

> El hábito más importante del segundo cerebro.
> Una idea no capturada es una idea perdida.

## Principio fundamental

Cuando capturas, **no organizas**.
Cuando organizas, **no capturas**.
Son dos modos mentales distintos — mezclarlos rompe ambos.

Todo va a `00_Inbox`. Sin pensar. Sin clasificar.

## Métodos de captura en Obsidian

### 1. Nueva nota rápida

Ctrl+N → escribe → guarda

La nota va automáticamente a `00_Inbox` 

Para que todas las notas creadas vayan a INBOX, asegurar de que está así configurado.
- Settings → Files & Links → Default location for new notes → 00_Inbox

### 2. QuickAdd (el más potente)
Plugin que permite capturar desde cualquier sitio
con un atajo de teclado sin interrumpir lo que haces.

#### Instalación:
Settings → Community plugins → QuickAdd Install → Enable

#### Configurción:
Settings → QuickAdd → Add Choice → Capture

- Name: Captura rápida
- Capture to: 00_Inbox/Inbox.md
- Format: ## {{DATE}} - {{VALUE}}
- Capture to active file: OFF

#### Asigna trabajo:

Settings → Hotkeys → QuickAdd: Run Captura rápida → Alt+C

Ahora con `Alt+C` aparece un popup,
escribes tu idea y desaparece. Sin salir de lo que hacías.

### 3. Desde móvil
Instala **Obsidian Mobile** y sincroniza con:
- Obsidian Sync (de pago) 
- Git + repositorio GitHub (gratis)
- iCloud / Google Drive apuntando al vault

La captura en móvil es igual — todo a Inbox.

## Qué capturar
- 💡 Ideas que aparecen de la nada
- 📖 Algo que aprendiste y no quieres olvidar
- 🔗 Links interesantes
- ✅ Tareas que surgen en el momento
- 💬 Reflexiones personales
- 📝 Notas de reuniones o conversaciones

## Qué NO hacer al capturar
- ❌ Buscar la carpeta correcta
- ❌ Añadir tags perfectos
- ❌ Escribir la nota perfecta
- ❌ Enlazar con otras notas
- ❌ Formatear con markdown

Todo eso va en la revisión semanal.

## Formato mínimo de captura
No necesitas más que esto:

```markdown
idea sobre X que se me ha ocurrido mientras corría
```

O si quieres un poco más de contexto:

```markdown
---
fecha: 2026-04-25
tags:
  - pendiente
---
idea sobre X — desarrollar luego
```

## El test de la captura
Una buena captura responde a:
**"¿Entenderé esto dentro de 2 semanas?"**

Si sí → suficiente.
Si no → añade una línea de contexto.
